# Introducción al Aprendizaje Automático con Meta AI

## 1. Paradigmas del ML: ¿Cuáles son las diferencias fundamentales entre el Aprendizaje Supervisado, No Supervisado y por Refuerzo? Para cada paradigma, proporciona un ejemplo de caso de uso real en la industria y explica qué tipo de datos y objetivos requiere cada uno.

**Paradigmas del Machine Learning: 3 formas distintas de aprender de los datos**

Los 3 paradigmas se diferencian principalmente por *qué tipo de feedback* usan para aprender y *qué problema* buscan resolver.

### 1. Aprendizaje Supervisado 
**Idea fundamental**: Aprende mapeando entradas a salidas usando ejemplos etiquetados. Es como estudiar con un profesor que te dice la respuesta correcta de cada ejercicio.

| Aspecto | Detalle |
| --- | --- |
| **Datos que requiere** | Datos etiquetados: cada ejemplo tiene un `input` y su `output` correcto. Ej: imagen → "gato" |
| **Objetivo** | Predecir o clasificar nuevos datos. Minimizar el error entre predicción y etiqueta real |
| **Feedback** | Directo e inmediato. El modelo sabe si se equivocó en cada ejemplo |
| **Subtipos principales** | Clasificación: predecir categorías. Regresión: predecir valores numéricos |

**Caso de uso real: Detección de fraude en banca**  
Bancos como BBVA o Nubank usan modelos supervisados para marcar transacciones sospechosas.  
- **Datos**: Millones de transacciones históricas etiquetadas como "fraude" o "legítima". Features: monto, ubicación, hora, dispositivo, historial del cliente.  
- **Objetivo**: Clasificar cada nueva transacción en 0.5 segundos como fraudulenta o no, para bloquearla antes de que se complete.  
- **Resultado**: Reducción de 60-80% en pérdidas por fraude.

### 2. Aprendizaje No Supervisado
**Idea fundamental**: Encuentra patrones ocultos en datos sin etiquetas. Es como que te den una caja de piezas de Lego sin instrucciones y tú agrupas por color o forma.

| Aspecto | Detalle |
| --- | --- |
| **Datos que requiere** | Datos NO etiquetados. Solo tienes los `inputs`, sin respuestas correctas |
| **Objetivo** | Descubrir estructura: agrupar, reducir dimensionalidad, detectar anomalías |
| **Feedback** | No hay "respuesta correcta". Se evalúa con métricas de cohesión interna |
| **Subtipos principales** | Clustering: agrupar. Reducción dimensional: simplificar. Reglas de asociación |

**Caso de uso real: Segmentación de clientes en e-commerce**  
Mercado Libre y Amazon lo usan para personalización sin saber de antemano qué tipos de clientes existen.  
- **Datos**: Comportamiento de navegación, historial de compras, clicks, tiempo en página. Sin etiquetas de "tipo de cliente".  
- **Objetivo**: Descubrir grupos naturales. Ej: "cazadores de ofertas", "compradores impulsivos tech", "clientes de carrito abandonado".  
- **Resultado**: Campañas de marketing dirigidas que aumentan conversión 15-30%. Cada segmento recibe recomendaciones distintas.

### 3. Aprendizaje por Refuerzo (RL)
**Idea fundamental**: Un agente aprende por prueba y error interactuando con un entorno. Recibe recompensas o castigos. Es como entrenar a un perro: no le dices qué hacer, solo premias el comportamiento bueno.

| Aspecto | Detalle |
| --- | --- |
| **Datos que requiere** | No un dataset estático. Necesita un entorno/simulador donde pueda tomar acciones y ver consecuencias |
| **Objetivo** | Maximizar recompensa acumulada a largo plazo. Aprender una política: qué acción tomar en cada estado |
| **Feedback** | Retrasado y escaso. La recompensa llega después de varias acciones. Ej: ganar o perder el juego |
| **Componentes clave** | Agente, entorno, acciones, estados, recompensa, política |

**Caso de uso real: Optimización de centros de datos en Google**  
DeepMind usó RL para controlar el sistema de enfriamiento de los data centers de Google.  
- **Datos**: Sensores en tiempo real: temperatura, presión, velocidad de ventiladores, carga de servidores, clima externo. No hay "respuesta correcta" etiquetada.  
- **Objetivo**: Minimizar consumo energético manteniendo temperatura segura. La recompensa = ahorro de energía.  
- **Resultado**: 40% menos energía usada en enfriamiento. El modelo descubrió estrategias que ningún ingeniero había probado, como aprovechar gradientes térmicos sutiles.

### Comparación rápida para decidir cuál usar

- **¿Tienes la respuesta correcta para tus datos históricos?** → **Supervisado**. Es el más común en industria, 80% de casos.  
- **¿Tienes muchos datos pero no sabes qué buscar en ellos?** → **No Supervisado**. Útil para exploración y segmentación.  
- **¿Tu problema es secuencial, de toma de decisiones, y puedes simularlo?** → **Refuerzo**. Más complejo de implementar, pero potente para control y juegos.

La clave: supervisado predice, no supervisado descubre, refuerzo actúa.


## 2. Redes Neuronales: ¿Qué es una red neuronal artificial y cómo se relaciona con el funcionamiento del cerebro humano? Explica qué son las épocas de entrenamiento, la función de pérdida y el descenso de gradiente, y cómo estos tres conceptos trabajan juntos para que un modelo "aprenda".

### **¿Qué es una red neuronal artificial?**

Una red neuronal artificial, o ANN, es un modelo matemático inspirado en cómo creemos que funciona el cerebro. No es una copia exacta, pero toma prestadas 3 ideas clave:

| Cerebro humano | Red neuronal artificial |
| --- | --- |
| **Neuronas** | Nodos que reciben entradas, hacen un cálculo y mandan una salida |
| **Sinapsis** | **Pesos**: números que determinan qué tan fuerte es la conexión entre nodos |
| **Aprendizaje** | Ajustar esos pesos para que la salida se parezca más a lo que queremos |

**Diferencia clave**: Tu cerebro tiene ~86 mil millones de neuronas, aprende con muy pocos ejemplos, consume 20W y es paralelo. Una red como GPT-4 tiene billones de parámetros, necesita millones de ejemplos, consume megawatts para entrenar, y corre en GPUs. La biología usa pulsos eléctricos y química. Las ANNs usan multiplicaciones de matrices y derivadas. 

Así que la relación es más metafórica que literal: tomamos la idea de "muchas unidades simples que juntas resuelven problemas complejos" y la hicimos funcionar con matemáticas.

### **Las 3 piezas que hacen que una red "aprenda"**

Para que una red aprenda necesita un ciclo: intentar, medir qué tan mal lo hizo, y corregir. Ahí entran estos 3 conceptos:

#### **1. Épocas de entrenamiento**
Una **época** = pasar por todo tu dataset de entrenamiento una vez completa.

Piénsalo como estudiar para un examen: 
- Leer todo el libro una vez = 1 época
- Si solo lo lees una vez, probablemente repruebes
- Si lo lees 50 veces, vas ajustando tu comprensión cada pasada

En ML, 1 época significa que la red ya vio cada ejemplo de entrenamiento al menos una vez y ajustó sus pesos. Modelos típicos necesitan 10 a 1000 épocas. Muy pocas épocas = *underfitting*, no aprendió. Demasiadas = *overfitting*, se memorizó el libro pero no entiende.

#### **2. Función de pérdida / Loss function**
La función de pérdida es el "examen" que califica qué tan mal lo está haciendo la red.

Toma la predicción de la red y la compara con la respuesta correcta. Devuelve un solo número: el **error**.  
- Error grande → la red está muy perdida  
- Error = 0 → predicción perfecta

**Ejemplos comunes**:
- **Error cuadrático medio** para regresión: $Loss = \frac{1}{n}\sum(y_{real} - y_{predicha})^2$ 
- **Entropía cruzada** para clasificación: castiga fuerte cuando dices "estoy 99% seguro que es gato" y era perro.

La pérdida es la brújula. Sin ella, la red no sabe si va bien o mal.

#### **3. Descenso de gradiente**
Si la función de pérdida te dice "qué tan mal estás", el descenso de gradiente te dice "hacia dónde corregir".

Imagina que estás perdido en una montaña con niebla y quieres llegar al punto más bajo del valle. No ves nada, pero puedes sentir la pendiente bajo tus pies. El descenso de gradiente hace eso:

1. Calcula el **gradiente**: la dirección de máxima subida del error respecto a cada peso
2. Da un paso en dirección contraria: `nuevo_peso = peso_viejo - learning_rate * gradiente`
3. Repite

El **learning rate** es el tamaño del paso. Muy grande → te pasas el valle y divergés. Muy chico → tardas una eternidad en bajar.

### **Cómo trabajan juntos: el ciclo de aprendizaje**

Todo se une en cada paso de entrenamiento así:

**Paso 1: Forward pass**  
Metes un batch de datos a la red. La info fluye hacia adelante y sale una predicción.

**Paso 2: Calcular pérdida**  
Comparas predicción vs realidad usando la función de pérdida. Obtienes un número: "te equivocaste por 2.7".

**Paso 3: Backward pass + Descenso de gradiente**  
Usas cálculo, específicamente *backpropagation*, para saber cuánto contribuyó cada peso al error de 2.7. El descenso de gradiente usa esa info para ajustar todos los pesos un poquito, de forma que el error baje la próxima vez.

**Paso 4: Repetir**  
Haces pasos 1-3 con todos tus datos. Cuando terminaste de ver todo el dataset, completaste **1 época**. 

Repites por 50 épocas. En cada época la pérdida debería bajar: época 1 loss=2.7, época 10 loss=0.8, época 50 loss=0.12. Cuando la pérdida deja de bajar, tu modelo "converge" y deja de aprender.

**Analogía completa**:  
Entrenar una red es como aprender a tirar al aro.  
- **Época**: Darle una vuelta completa a todos tus tiros de práctica  
- **Función de pérdida**: Medir cuántos cm te quedó lejos del aro cada tiro  
- **Descenso de gradiente**: Ajustar el ángulo de tu brazo y fuerza basado en si te fuiste largo o corto  

Después de 1000 tiros, tu cerebro ajustó la "musculatura" y ahora encestas. La red ajustó sus "pesos" y ahora predice bien.


## 3. Scikit-learn vs TensorFlow: ¿Cuáles son las principales diferencias entre Scikit-learn y TensorFlow como frameworks de Machine Learning? ¿En qué situaciones es más conveniente usar cada uno, y existe algún caso en que sea recomendable utilizarlos juntos en un mismo proyecto?

**Scikit-learn vs TensorFlow: No compiten, se complementan**

Son herramientas para problemas distintos. Pensalo como un martillo vs una fábrica completa de herramientas.

### **Diferencias fundamentales**

| Aspecto | Scikit-learn | TensorFlow |
| --- | --- | --- |
| **Nivel de abstracción** | Alto. `model.fit(X, y)` y listo | Bajo. Tú defines la arquitectura, el loop de entrenamiento, todo |
| **Tipo de modelos** | ML clásico: Regresión, SVM, Random Forest, K-Means, PCA | Deep Learning: Redes neuronales densas, CNNs, RNNs, Transformers |
| **Ejecución** | Solo CPU, en memoria. Un solo proceso | CPU/GPU/TPU. Distribuido. Grafos computacionales |
| **Datos que maneja bien** | Tabulares, datasets pequeños/medianos <10M filas | Imágenes, audio, texto, video. Datasets masivos |
| **Flexibilidad** | Limitada. Usas algoritmos pre-hechos | Total. Construyes cualquier arquitectura desde cero |
| **Curva de aprendizaje** | 1 día para ser productivo | Semanas para entender bien tensores, grafos, backprop |
| **Código típico** | 10 líneas para entrenar un modelo | 100+ líneas para lo mismo, pero con control total |

**Analogía**: Scikit-learn es una caja de herramientas IKEA: agarras un RandomForest, lo ensamblas en 5 min y funciona. TensorFlow es el taller completo: puedes construir tu propio mueble desde la madera, pero toma tiempo y skill.

### **¿Cuándo usar cada uno?**

#### **Usa Scikit-learn cuando:**
1. **Tus datos son tabulares**: CSVs, SQL, Excel. Filas y columnas. Ej: predecir churn, riesgo crediticio, precios de casas
2. **Dataset < 1GB y cabe en RAM**: Si `df.memory_usage().sum()` < 16GB, sklearn vuela
3. **Necesitas velocidad de desarrollo**: Baseline en 30 min. Ideal para hackathons y POCs
4. **Modelos interpretables**: Necesitas explicar por qué el modelo decidió. `feature_importances_` de un Random Forest es fácil de explicar a negocio
5. **Preprocesamiento robusto**: `Pipeline`, `ColumnTransformer`, `OneHotEncoder`, validación cruzada. Todo integrado

**Caso real**: Un banco predice si aprueba un crédito. Datos: 50 columnas, 500k clientes. RandomForest de sklearn en 2 min de entrenamiento, AUC 0.91, y puedes explicar que "ingreso" y "deuda actual" son las variables top. Ganaste.

#### **Usa TensorFlow cuando:**
1. **Datos no estructurados**: Imágenes, texto, audio, series de tiempo largas. Ej: detectar tumores en rayos X, traducir idiomas
2. **Necesitas Deep Learning**: Ningún modelo de sklearn le gana a una ResNet en visión por computadora
3. **Datasets gigantes**: 100M de imágenes. TensorFlow + GPUs + `tf.data` para streaming
4. **Quieres producción a escala**: TensorFlow Serving, TFLite para móviles, TF.js para web
5. **Investigación / arquitecturas custom**: Si estás inventando la próxima arquitectura de transformers, necesitas el control de bajo nivel

**Caso real**: Tesla Autopilot. Input: 8 cámaras de video en tiempo real. Output: segmentación + detección de objetos + path planning. Eso es 99% redes neuronales. Sklearn ni siquiera compila ahí.

### **¿Se pueden usar juntos? Sí, y es muy común**

El patrón más usado en industria es: **Scikit-learn para todo, TensorFlow solo para la parte que lo necesita**.

**Pipeline híbrido típico:**

1. **Preprocesamiento con Sklearn**  
   `StandardScaler`, `OneHotEncoder`, `KNNImputer`. Sklearn es imbatible para esto. TensorFlow tiene `tf.feature_column` pero es más verboso.

2. **Feature engineering con Sklearn**  
   `PCA`, `SelectKBest`, `PolynomialFeatures`. Generas nuevas columnas antes de meterlas a la red.

3. **Modelo base con Sklearn para benchmark**  
   Entrenas un `XGBoost` en 5 min. Si te da 0.85 AUC, ya sabes que tu red neuronal tiene que superar eso para valer la pena.

4. **Deep Learning con TensorFlow/Keras**  
   Si los datos son imágenes o texto, aquí entra TF. O si el baseline de sklearn se quedó corto.

5. **Post-procesamiento y evaluación con Sklearn**  
   `classification_report`, `confusion_matrix`, `cross_val_score`, `GridSearchCV`. Incluso puedes envolver un modelo de Keras en `KerasClassifier` para usarlo dentro del ecosistema sklearn.

**Ejemplo concreto: Clasificación de reseñas de Amazon**

```python
# 1. Sklearn para preprocesamiento
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.model_selection import train_test_split
vectorizer = TfidfVectorizer(max_features=5000)
X_tfidf = vectorizer.fit_transform(textos)

# 2. Sklearn para baseline rápido

print("Baseline sklearn:", baseline.score(X_test_tfidf, y_test)) # 0.88

# 3. TensorFlow si necesitas superarlo

    tf.keras.layers.Embedding(10000, 64),
```

**Regla de oro**: Empieza siempre con Scikit-learn. Si resuelve tu problema con 90% de la calidad y 10% del esfuerzo, quedate ahí. Solo salta a TensorFlow cuando los datos o el problema te obligan.
